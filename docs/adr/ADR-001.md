# ADR-001: Clean Architecture + DDD sobre MVC estándar de Laravel

- **Estado**: Aceptado
- **Fecha**: 2026-06-19

## Contexto
El MVC estándar de Laravel mezcla lógica de negocio, persistencia e HTTP en controladores y modelos Eloquent, lo que dificulta el mantenimiento y testeo en dominios complejos como el de Urbania.

## Decisión
Implementar una arquitectura de 4 capas (Domain, Application, Infrastructure, Presentation) con bounded contexts aislados siguiendo Domain-Driven Design. Las dependencias apuntan hacia adentro: Presentation → Application → Domain, con Infrastructure implementando interfaces definidas en capas internas.

## Consecuencias
- ✅ Separación clara de responsabilidades
- ✅ Testeabilidad: Domain y Application no requieren DB real
- ✅ Bounded contexts independientes (ej. Auth no conoce Billing)
- ❌ Mayor cantidad de archivos y estructura más compleja
- ❌ Curva de aprendizaje para desarrolladores acostumbrados a MVC estándar
