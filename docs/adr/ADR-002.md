# ADR-002: RS256 sobre HS256 para firma de JWT

- **Estado**: Aceptado
- **Fecha**: 2026-06-19

## Contexto
Laravel por defecto usa HMAC-SHA256 (HS256) para firmar tokens JWT, compartiendo un secreto simétrico. En una arquitectura de microservicios o con múltiples consumidores, esto es inseguro porque cualquier servicio que conozca el secreto puede generar tokens válidos.

## Decisión
Usar RS256 (RSA-SHA256) con un par de claves RSA de 4096 bits. La clave privada firma los tokens (solo la API la conoce) y la clave pública los verifica (puede distribuirse a otros servicios).

## Consecuencias
- ✅ Mayor seguridad: la clave privada nunca sale del servidor de auth
- ✅ Compatibilidad con arquitecturas distribuidas y microservicios
- ✅ Rotación de claves independiente
- ❌ Mayor overhead computacional en generación de tokens
- ❌ Tamaño de token ligeramente mayor
