# ADR-004: Doble token (Access + Refresh) con rotación

- **Estado**: Aceptado
- **Fecha**: 2026-06-19

## Contexto
Un solo token JWT de larga duración, si es robado, otorga acceso indefinido hasta su expiración. Separar la autenticación en dos tokens con diferentes vidas útiles y responsabilidades permite limitar el daño de un token comprometido.

## Decisión
Access token JWT (RS256, 15 min) para autorizar requests + Refresh token opaco (hash SHA-256 en DB, 14 días) para renovar el access token. El refresh token rota en cada uso: el anterior se revoca, el nuevo se emite. Si se detecta reutilización de un refresh token ya rotado, se revoca toda la familia de tokens del usuario.

## Consecuencias
- ✅ Robo de access token tiene ventana limitada de 15 minutos
- ✅ Detección de replay: reutilización → revocación masiva + evento de seguridad
- ✅ Revocación granular por sesión (cada login crea una sesión independiente)
- ❌ Mayor complejidad en cliente (debe manejar rotación de refresh tokens)
- ❌ Latencia adicional en cada refresh (consulta a DB + Redis blacklist)
