# ADR-005: Pest sobre PHPUnit directo

- **Estado**: Aceptado
- **Fecha**: 2026-06-19

## Contexto
PHPUnit, aunque potente, tiene una sintaxis verbosa heredada de JUnit (Java) con clases que extienden TestCase y métodos con prefijo `test`. Para un proyecto que busca legibilidad y expresividad en tests, esto añade ruido innecesario.

## Decisión
Usar Pest como framework de testing principal. Pest envuelve PHPUnit y mantiene compatibilidad total con sus assertions, mocks, data providers y configuración, pero ofrece una API más concisa con funciones `it()`, `describe()`, `expect()`, y tests de arquitectura nativos.

## Consecuencias
- ✅ Tests más legibles y concisos (menos boilerplate)
- ✅ Arquitectura de tests nativa (verificar dependencias entre capas)
- ✅ 100% compatible con PHPUnit (assertions, mocks, data providers)
- ❌ Curva de aprendizaje para desarrolladores con experiencia solo en PHPUnit
- ❌ Menor ecosistema de plugins que PHPUnit
